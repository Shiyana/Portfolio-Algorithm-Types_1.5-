Algorithm Catalog

Each algorithm below includes its learning type, main application domains, a simple explanation of how it works, and an example use case. Together, these models illustrate how AI systems vary depending on the data structure, task, and expected output.

Tabular Data Computer Vision NLP Generative AI

1. Linear Regression

Supervised
Tabular Data

How it works: Models the relationship between input variables and a continuous output by fitting a best-fit line.

Use case: Predicting housing prices, demand forecasting, and revenue estimation.

2. Decision Tree

Supervised
Tabular Data

How it works: Splits data into branches based on feature rules until it reaches a prediction or class label.

Use case: Loan approval, customer churn prediction, and medical decision support.

3. Random Forest

Supervised
Tabular Data

How it works: Combines many decision trees and averages their predictions to improve accuracy and reduce overfitting.

Use case: Fraud detection, credit scoring, and healthcare risk prediction.

4. Support Vector Machine (SVM)

Supervised
Tabular Data Computer Vision

How it works: Finds the optimal boundary that separates classes with the maximum margin.

Use case: Email spam classification, handwriting recognition, and image categorization.

5. K-Means Clustering

Unsupervised
Tabular Data

How it works: Groups similar data points into clusters by minimizing distance from each cluster center.

Use case: Customer segmentation, behavior analysis, and anomaly exploration.

6. Hierarchical Clustering

Unsupervised
Tabular Data

How it works: Builds a tree of clusters by iteratively merging or splitting groups based on similarity.

Use case: Gene expression analysis, market segmentation, and document grouping.

7. Artificial Neural Network (ANN)

Supervised / Deep Learning
Computer Vision NLP

How it works: Uses layers of interconnected neurons to learn complex, nonlinear relationships in data.

Use case: Pattern recognition, sentiment classification, and predictive analytics.

8. Convolutional Neural Network (CNN)

Supervised / Deep Learning
Computer Vision

How it works: Uses convolutional filters to detect spatial features like edges, textures, and objects in images.

Use case: Facial recognition, autonomous driving, and medical image diagnosis.

9. Transformer Models

Deep Learning
NLP Generative AI

How it works: Uses attention mechanisms to understand context and relationships across sequences of text or tokens.

Use case: Chatbots, machine translation, summarization, and large language models.

10. Generative Adversarial Network (GAN)

Generative Deep Learning
Generative AI Computer Vision

How it works: Trains a generator and discriminator together so the model learns to create realistic synthetic data.

Use case: AI-generated images, synthetic training data, style transfer, and media generation.

Visual Framework

This framework shows how algorithms can be grouped first by learning style and then mapped to the AI domains where they are most commonly used. The diagrams below are designed to make the relationships between algorithm families, data types, and real-world tasks immediately visible.

Learning Style Categories

Supervised Learning
Trains on labeled data to predict outputs or classify observations.
Examples: Linear Regression, Decision Tree, Random Forest, SVM
Unsupervised Learning
Finds patterns and hidden structures in unlabeled data.
Examples: K-Means, Hierarchical Clustering
Deep Learning
Uses multi-layer neural networks for highly complex tasks involving images, text, and generation.
Examples: ANN, CNN, Transformers, GANs

Primary Application Domains

Tabular Data
Structured rows and columns used in forecasting, finance, operations, and business analytics.
Computer Vision
Image and video understanding, including object recognition, diagnosis, and scene analysis.
Natural Language Processing
Text and language understanding for translation, summarization, sentiment, and conversation.
Generative AI
Models that create new content such as text, images, audio, or synthetic data.

Portfolio Reflection

Creating this portfolio artifact strengthened my understanding of how machine learning algorithms differ by learning style, domain, and practical purpose. Before organizing this framework, I understood many algorithms individually, but mapping them across tabular data, computer vision, NLP, and generative AI made their differences much clearer. I saw more clearly that traditional supervised models such as linear regression, decision trees, and random forests are especially effective for structured business data, while deep learning models such as CNNs and transformers are more suitable for image and language tasks.

I also gained a stronger appreciation for generative AI as a distinct and rapidly growing application space. Transformer-based models and GANs do more than analyze data; they create new content, which makes them especially important in modern AI development. Building this page helped me communicate technical ideas in a more visual and professional way, which is valuable for both academic work and career growth. This artifact demonstrates not only my knowledge of algorithms, but also my ability to organize, explain, and present AI concepts clearly for a portfolio audience.
